OS-消息传递

Created2022-08-06|操作系统

实验七报告

实验代码

验证

原理性解释

  • 通过msgsnd和msgrcv程序实现两个进程之间的消息传递。
  • 在msgsnd中根据输入的文件名生成的key创建一个消息队列，然后讲msgsnd.txt中的内容写入消息队列中，而在msgrcv中根据输入相同的文件名找到msgsnd进程创建的消息队列，再根据输入的消息号从消息队列中读取相应的信息，同时，两个程序结合在终端执行ipcs -q得到的消息队列的信息进行比较验证。
  • 主要的系统调用如下：
    • key_t ftok(char *pathname, char proj_id)
      • 与共享内存中所用的ftok相同，作用为用传入的带路径的文件名pathname及选定的id生成一个键值
    • int msgget(key_t key, int msgflg)
      • 用来创建新的消息队列或获取已有的消息队列
      • key：消息队列对象的关键字，数将它与已有的消息队列对象的关键字进行比较来判断消息队列对象是否已经创建
      • msgflg：操作控制标识，可取如下几个值
        • IPC_CREAT：如果消息队列对象不存在，则创建之，否则则进行打开操作;
        • IPC_EXCL：和IPC_CREAT一起使用，如果消息对象不存在则创建之，否则产生一个错误并返回。
        • 此外还有一些存取权限控制符，与文件系统中的权限控制符类似。
      • 返回值：如果单独使用IPC_CREAT标志，msgget()函数要么返回一个已经存在的消息队列对象的标识符，要么返回一个新建立的消息队列对象的标识符。如果将IPC_CREAT 和IPC_EXCL标志一起使用，msgget()将返回一个新建的消息对象的标识符，或者返回-1 如果消息队列对象已存在。
    • int msgsnd(int msgqid, const void *msgp, size_t msgsz, long msgtyp, int msgflg)
      • 作用为将一条消息加到消息队列中
      • msqid: 消息队列的标识符
      • msgp: 消息缓冲区指针
      • msgsz: 消息数据的大小
      • msgtyp：消息类型
      • msgflg：消息发送方式，０表示阻塞方式，IPC_NOWAIT表示非阻塞方式
      • 返回值：调用成功返回0，不成功返回-1
    • ssize_t msgrcv(int mqsqid, const void *msgp, size_t msgsz, long msgtyp, int msgflg)
      • 作用为将一条消息加到消息队列中
      • msqid: 消息队列的标识符
      • msgp: 消息缓冲区指针
      • msgsz: 消息数据的大小
      • msgtyp：消息类型
      • msgflg：消息发送方式，０表示阻塞方式，IPC_NOWAIT表示非阻塞方式
      • 返回值：调用成功返回0，不成功返回-1
    • int msgctl(int msqid, int cmd, struct msqid_ds *buf)
      • msqid: 消息队列的标识符
      • cmd：对消息队列进行的操作
        • IPC_STAT：取出系统保存的消息队列的msqid_ds数据，并将其存入参数buf指向的msqid_ds结构中。
        • IPC_SET：设定消息队列的msqid_ds数据中的msg_perm成员。设定的值由buf指向的msqid_ds结构给出。
        • IPC_RMID：将队列从系统内核中删除。
      • buf：执行系统消息队列结构体的指针。

细节解释

Alg.9-0-msgdata.h

  • 首先，宏定义消息大小TEXT_SIZE的值为512，由于默认最大的队列大小为16384bytes，故即定义了队列结点为16484/512 = 32个
  • 其次，定义了消息的结构体msg_struct，其中有一个长整型变量msg_type表示消息的类型，以及一个字符数组mtext用于存放消息内容。
  • 然后是与前几个实验相同的宏定义——PERM和ERR_EXIT

Alg.9-1-msgsnd.c

  • 头文件作用：
    • : 提供了消息队列的相关函数
  • 首先，与前面的实验一样，先读取传入main函数的参数，作为文件路径名，然后通过ftok()函数获取key值
  • 获得key值之后将其输出并调用msgget()函数建立一个消息队列并返回其id值，存在msqid中
  • 接着通过fopen()函数打开要写进消息队列的文件msgsnd.txt
  • 定义消息队列信息结构体变量msqattr，然后通过msgctl()函数将系统保存的消息队列的msqid_ds数据存放入msqattr中，接着便可输出当前消息队列剩余的消息以及位置
  • 然后读取txt文件中每一条消息的消息类型及内容，打印出来后通过msgsnd()函数发送到msqid对应的消息队列中，由于msgsnd()函数最后一个参数选择了0，因此为阻塞发送，即当队列满之后会停下来等待
  • 最后输出一共发送的消息数，关闭txt文件并通过system()函数在中断执行ipcs -q指令查看消息队列的相关信息

Alg.9-2-msgrcv.c

  • 首先同样先读取传入main函数的第一个参数，作为文件路径名，然后通过ftok()函数获取key值
  • 然后通过msgget()函数打开在msgsnd.c中创建好的消息队列，注意到msgget()函数并没有加上参数IPC_CREAT，保证了只会打开已存在的消息队列而不会创建。
  • 然后读取传入main函数的第二个参数，作为消息类型
  • 接着通过一个循环，不断调用msgrcv()函数接收消息队列中的消息并输出，注意到由于msgrcv()函数最后一个参数为IPC_NOWAIT，因此为非阻塞接收，即当消息队列为空时msgrcv()函数将不会等待，而是返回-1
  • 当接收完毕之后，同样通过调用msgctl()函数将系统保存的消息队列的msqid_ds数据存放入msqattr中，然后输出消息队列中剩余的消息数
  • 若消息数为0，则询问是否删除消息队列，若选择删除则会通过调用msgctl()函数，传入参数IPC_RMID将该消息队列删除

运行结果

  • 先运行msgsnd.c程序，可以看到建立了一个消息队列，其中有10条消息
  • 然后再运行msgrcv.c程序，输入参数2表示读取消息类型为2的消息，可以看到读出了两条消息类型为2的消息，然后消息队列中剩余8条消息
  • 再次运行msgrcv.c程序，输入参数0表示读取所有消息，可以看到剩余的8条消息均被读出，最后消息队列中剩余0条消息
  • 再选择删除消息队列，最后执行ipcs -q可以看到已无消息队列存在
  • 重新运行msgsnd.c程序创建消息队列，并再次运行msgrcv.c程序，输入参数0，可以看到10条信息被读出，选择不删除消息队列，可以看到最后消息队列仍存在，但其消息数为0

设计

消息传递机制的相关POSIX函数

  • 首先有一个结构体变量mq_attr，其内容如下：

    可通过函数getattr(mqID,&mqAttr)，获取mqID对应的消息队列的属性。

  • mqd_t mq_open(const char name, int oflag, mode_t mode, struct mq_attr attr)
    • 创建或打开一个消息队列
    • 返回值为该消息队列对应的ID
    • name：文件路径名，以/开头
    • oflag：访问(receive/send)消息队列的方式，具体参数有
      • O_RDONLY：只读，打开一个消息队列用来接受消息
      • O_WRONLY：只写，打开一个消息队列用来发送消息
      • O_RDWR：读写，打开一个消息队列用即可用来发送消息也可以用来接受消息
      • O_CREAT：创建，若使用这个参数需要追加后面的2个参数。
      • O_EXCL：既存检查，检查消息队列是否已经存在
      • O_NONBLOCK：非阻塞，若没有加此参数，默认是阻塞形式
    • 若name参数指定的消息队列已经存在, 那么后面两个参数将不起任何作用
    • 若不存在且O_CREAT被设置
      • mode：表示访问权限
      • attr：给新队列设定某些属性，如果是空指针，那么就采用默认属性
  • mqd_t mq_close(mqd_t mqdes)
    • 断开消息队列描述符和与之对应消息队列间的连接
    • mqdes：消息队列的描述符
    • 如果函数执行成功, 函数返回0; 如果函数执行失败,函数返回-1
  • mqd_t mq_unlink(const char* name)
    • 从内核中删除名为name的消息队列(如果该函数被调用了,但是仍然有进程已经打开了这个消息队列,那么这个消息队列的销毁会被推迟到所有的引用都被关闭时执行)
    • 如果函数执行成功, 函数返回0; 如果函数执行失败,函数返回-1
  • mqd_t mq_send(mqd_t mqdes, const char *msg_ptr, size_t msg_len, unsigned msg_prio)
    • 将参数 msg_ptr 指向的内容发送给参数mqdes 指向的消息队列
    • mqdes：要发送的消息队列的描述符
    • msg_ptr：指向消息结构体的指针
    • msg_len：消息的字节数，不能大于消息队列的属性 mq_msgsize 的值
    • msg_prio：优先级大的消息会插队在优先级小的消息前面，同优先级的消息会根据时间先后排队
    • 如果函数执行成功, 函数返回0; 如果函数执行失败,函数返回-1
  • mqd_t mq_receive(mqd_t mqdes, char msg_ptr, size_t msg_len, unsigned msg_prio)
    • 从 mqdes 指定的消息队列中取出最高优先级中排队时间最久的消息，如果参数 msg_len 比消息队列的属性mq_msgsize小, 函数会执行失败，函数执行成功后,选定的消息会拷贝到参数msg_ptr指定的内存中并且从消息队列中删除
    • mqdes：要接收的队列的描述符
    • msg_ptr：指向消息结构体的指针, 存放接收到的消息
    • msg_len：消息的字节数, 不能小于 mq_msgsize
    • msg_prio：作为返回值，存放接收的消息的优先级
    • 如果函数执行成功, 函数返回0; 如果函数执行失败,函数返回-1

msgcon.c

  • 仿照alg.8-4~8-6，利用一个控制台程序生成两个子进程来实现进程间的消息传递机制。msgcon.c就是这样的一个控制台程序，代码如下：
  • 第一部分是变量的定义以及传入main函数参数的判断，这与前面的程序大同小异
  • 第二部分首先是根据传入main函数的文件路径名调用mq_open()函数以创建一个消息队列，注意到此时为首次以该文件名打开，故肯定需要创建，故需要使用参数O_CREAT，0666表示文件的访问权限，NULL表示使用默认的消息队列属性，创建成功之后会将该队列的id传到msqid中。调用之后是一个错误判断
  • 接着是利用mq_getattr()函数获取创建成功的队列的信息，存放在mq_attr结构体变量mqAttr中，然后输出mqAttr.mq_curmsgs，即当前消息队列的消息数
  • 然后利用vfork()和execv()函数运行编译好的msgsnd.o开启第一个子进程，即消息发送进程，同时将文件路径名传入其中
  • 紧接着再次利用vfork()和execv()函数运行编译好的msgrcv.o开启第二个子进程，即消息接收进程，同时将文件路径名传入其中
  • 第三部分则是产生子进程之后父进程的工作，即先通过wait()等待两个子进程的结束
  • 当两个子进程均结束以后，即代表消息的发送和接收均结束，因此通过mq_close()解除链接并通过mq_unlink()删除消息队列

msgsnd.c

  • 此程序为消息发送子进程的实现
  • 第一部分是变量的定义以及传入main函数参数的判断，若参数正确则将其传入pathname中
  • 第二部分是根据pathname利用mq_open()函数打开在msgcon.c已经建好的消息队列，注意此时不需要加参数O_CREAT，并且消息的接收选择为非阻塞接收，故需要参数O_NONBLOCKING,0666表示文件的访问权限，NULL表示使用默认的消息队列属性，创建成功之后会将该队列的id传到msqid中。调用之后是一个错误判断
  • 第三部分首先是先通过fopen()打开txt文件，然后逐行读取文件中的消息内容和消息优先级
  • 然后通过mq_send()函数将读取的消息写入到前面打开的消息队列中
  • 每次写入之后通过mq_getattr()函数获取当前消息队列的信息并将其输出
  • 最后文件读取结束并写入完成之后便输出写入的消息数然后结束进程

msgrcv.c

  • 此程序为消息接收子进程的实现
  • 第一部分是变量的定义以及传入main函数参数的判断，若参数正确则将其传入pathname中
  • 第二部分首先根据pathname利用mq_open()函数打开在msgcon.c已经建好的消息队列，注意此时不需要加参数O_CREAT，并且消息的发送选择为非阻塞发送，故需要参数O_NONBLOCKING,0666表示文件的访问权限，NULL表示使用默认的消息队列属性，创建成功之后会将该队列的id传到msqid中。调用之后是一个错误判断
  • 然后先通过mq_getattr()获取消息队列的信息，以便后续调用mq_receive()
  • 调用mq_receive()接收消息之后将消息的内容和优先级分别输出
  • 每次读出之后通过mq_getattr()函数获取当前消息队列的信息并将其输出
  • 由于设置接收为非阻塞接收，故当最后消息队列为空之后，则会输出共接收的消息数量并退出循环结束进程

运行结果：

非并发执行

  • 首先是输出该消息队列的ID，其次是在msgcon.c中输出消息队列的消息数，由于此时还未开启发送进程，消息队列刚初始化，故消息数自然是为0
  • 可以看到发送进程依次读取txt文件内的内容并发送，消息队列中的消息数也便由0增加到10
  • 由于在接收进程中一开始调用了sleep(1)，故会等到全部消息发送完才会开始接收，从输出的所接收的优先级可以看到，接收的规则为优先接收优先级高且发送时间较早的消息。每接收一条消息，队列消息数则会减一，直到队列为空，子进程结束。
  • 父进程等到了两个子进程都结束之后便删除消息队列结束进程。

并发执行

  • 若在发送进程也加了sleep()，则可实现发送接收的并发执行
  • 一下结果为发送和接收均调用sleep(1)
  • 可以看到输出的结果出现了交错，但大致为往消息队列中写入一条之后便读出一条
