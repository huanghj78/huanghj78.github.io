alg.13-1-pthread-create-1，-2

原理性解释

  • 这几份程序代码提供了最基本的多线程编程的框架，通过在主线程中创建一个子线程并通过参数传递或共有数据的共享实现信息传递，以实现子线程完成主线程的一定任务。
  • 引用的头文件主要为：
    • < pthread >：是POSIX的一个库，提供了线程相关的函数，故在编译的时候需加上“-pthread”
  • 用到的主要函数为：
    • int pthread_attr_init(pthread_attr_t *attr)
      • 功能为初始化一个线程属性对象
      • 参数为一个pthread_attr_t类型的地址，pthread_attr_t是一个结构体，其中包含了线程的一些属性，内容如下
  • int pthread_create(pthread_t * tidp, const pthread_attr_t * attr, void * ( * start_rtn)(void*),void *arg);
    • 功能为：创建线程（实际上就是确定调用该线程函数的入口点），在线程创建以后，就开始运行相关的线程函数。
    • 第一个参数为指向线程标识符的指针。
    • 第二个参数用来设置线程属性，若选择NULL则为默认属性。
    • 第三个参数是线程运行函数的起始地址。
    • 最后一个参数是运行函数的参数。
    • 需要注意的是这四个参数需要传入的都是地址
    • 返回值：若线程创建成功，则返回0。若线程创建失败，则返回出错编号
  • int pthread_join(pthread_t thread, void **retval)
    • 功能为：以阻塞的方式等待thread指定的线程结束。当函数返回时，被等待线程的资源被收回。
    • 第一个参数为等待线程的标识符。
    • 第二个参数为用户定义的指针，用来存储被等待线程的返回值。
    • 返回值：0代表成功。 失败，返回的则是错误号。
  • void pthread_exit(void * value_ptr)
    • 实际上相当于return value_ptr
    • 功能为：结束线程并返回某个类型的变量到调用的父线程，后者通过pthread_join函数中的第二个参数得到返回值
    • value_ptr为指向要返回的值的指针

细节解释

alg.13-1-pthread-create.c

  • 此程序通过一个全局变量sum作为线程之间的共享变量以实现线程之间的消息传递。
  • 首先，定义一个线程函数runner，即，生成的子线程将会运行的函数，其被声明为static，目的是为了不会被外部文件调用，返回类型和传入参数均为void*，目的是为了能够通过类型转化返回或传入各种类型的数据。
  • 在此程序中，runner函数的主要内容是根据传入的一个整数，计算1到该整数的和，并将结果赋值给全局变量sum，这样一来，主线程就可以得到子线程的结果。runner函数中要使用传入的参数一般要先进行类型转化，如此代码中，将void*类型的实参param通过一个函数atoi()转化为整数赋值给upper。
  • 在完成子线程的任务之后，调用pthread_exit()函数，即可结束子线程并按传入的参数决定是否向主线程返回内容。
  • 在main函数，即主线程中，通过pthread_attr_init()函数初始化线程属性之后调用pthread_create()函数创建了子线程并将要求和的上限传入到子线程，此时子线程便开始执行，然后主线程调用pthread_join()函数，阻塞等待子线程的结束。
  • 最终主线程打印出全局变量sum的值，即，子线程的计算结果。

alg.13-1-pthread-create-1-1.c

  • 此程序与上一个程序的区别在于在上一个的基础上，在线程函数中通过malloc函数动态申请了一个int变量，并将其作为子线程的返回值通过pthread_exit()函数返回到主线程，主线程通过pthread_join()函数接收并输出从而验证了此方法能够正确实现两个线程之间的信息传递。
  • 线程函数中定义了一个整型指针retptr指向动态申请的整型变量空间，然后在返回的时候需要强制类型转换为void*，同时，在主线程接收的时候也需要注意先进行强制类型转换。
  • 需要注意的是用于线程之间传递的变量需要为全局有效的变量，如上一个程序所用到的全局变量，或在此程序中通过动态申请在堆上的变量，或后面程序用到的主线程中的局部变量，而不能是线程函数中的局部变量，因为局部变量在线程函数结束之后便由于弹栈而被销毁。同时，由于是在堆上的变量，在主线程中获取之后需要再调用free()函数进行释放。

alg.13-3-pthread-stack.c

原理性解释

  • 此程序用到一个新的函数pthread_att_setstack()，用以设定线程栈的地址和大小。
  • 此程序的目的是验证线程栈的大小能否接近达到1.9G，通过不断地递归线程函数直到发生段错误，即线程栈满了，统计递归的次数，结合每次递归所需要的空间即可大致地算出线程栈的大小。

细节解释

  • 首先，在线程函数test中定义一个大小为1024个字节的字符数组buffer，以此大小视为每次递归所需的空间大小。然后不断地进行递归。
  • 在主函数中则是先利用pthread_att_setstack()设置线程栈的大小为1.9G，然后通过pthread_create()函数创建子线程开始递归直到段错误，最后根据递归次数便可算出线程栈大小。

alg.13-4-openmp-demo.c

原理性解释

  • 此程序是利用opem mp（用于共享内存并行系统的多线程程序设计的一套编译指令）来实现多线程编程，语法为：#pragma omp parallel num_threads(n)以生成n个线程或#pragma omp parallel默认生成特点数量线程，对于Intel的CPU每个核对应生成两个线程
  • 其中#pragma表示的是预处理指令

细节解释

  • 此程序通过依次传入不同的数值测试生成不同数量的线程
  • 运行结果如下：


    可以看到，由于虚拟机只分配了一个核，调用#pragma omp parallel只默认生成了两个线程。且当#pragma omp parallel num_threads(2)执行完毕之后，执行#pragma omp parallel num_threads(4)生成的四个线程的tid跟前面的tid有相同的情况，原因是前面的线程结束之后，tid实现了复用。

alg.13-5-openmp-matrixadd.c

原理性解释

  • 此程序利用open mp实现多线程的矩阵相加计算，并比较不同的线程数所耗费的时间。

细节解释

  • 首先定义一个正常的单线程矩阵相加的函数。
  • 结合gettimeofday()函数可以实现计时，依次执行单线程的计算，以及通过#pragma omp parallel num_threads(n)进行多线程计算，计算各自耗费的时间输出进行比较。
  • 比较不同矩阵阶数计算的效率，运行结果如下：

    可以看到当阶数较小时，多线程的计算效率反倒比较低，原因是多线程需要额外的系统开销，而当阶数较大时，多线程的计算效率便高于单线程（由于虚拟机只分配一个核，故无法实现四个线程的并发，因此四个线程的时候反倒更慢）

alg.13-6-fork-pthread-demo1.c

原理性解释

  • 此程序结合了pthread_create()和fork()，在主线程中调用pthread_create()生成一个子线程执行线程函数thread_worker()，然后再在主线程调用fork()生成一个子进程，在父进程和子进程中分别执行“ps -l -T”查看进程信息。

细节解释

  • 首先定义线程函数，其实现的是不断地打印全局变量i
  • 然后在主函数中，先pthread_create函数开启子线程，再调用fork开启子进程，子进程执行“ps -l -T”显示进程信息，然后父进程通过wait()等待子进程的结束后也执行“ps -l -T”显示进程信息，然后通过while(1)进入死循环。
  • 运行结果如下：

    在子进程中可以看到执行a.out生成的三个任务，前两个的PPID均为1856，即为bash进程的PID，因此前两个是主进程以及主进程生成的子线程，而第三个的PPID为19526，即为主进程生成的子进程。
    而由于主进程执行了wait(&p)，因此当子进程结束之后父进程才会执行“ps -l -T”显示进程信息，故在父进程中看到执行a.out只生成了两个任务，即主进程以及主进程生成的子线程。
    由于子线程中是一个死循环，故会一直打印0，需要使用pkill -f a.out结束该进程。

alg.13-7-fork-pthread-demo2.c

原理性解释

  • 此程序与上一个程序不同在于上一个程序是在主函数中进行fork，而此程序是在线程函数，即子线程中进行fork。

细节解释

  • 首先定义线程函数，其实现的是调用fork()函数生成一个子进程，在子进程中执行的是修改全局变量i的值由0改为1，并执行“ps -l -T|grep a.out”查看进程信息。接下来是一个死循环不断地打印全局变量i的值，因此，主进程会打印0而子进程打印1.
  • 主函数中生成子线程sleep了两秒之后执行“ps -l -T”显示进程信息，然后执行return 1结束了进程。因此子进程无法结束，会不断地打印“1”

    spid=19648，pid=19647为主进程创建的子线程
    spid=19649，pid=19649为子进程

    由于父进程和子进程都没有结束，因此会不断地打印0和1

alg.13-8-sigaction-demo.c

原理性解释

  • 用到的一个新的函数sigaction()
    • int sigaction(int signum, const struct sigaction *act,
      struct sigaction *oldact)
      • signum参数指出要捕获的信号类型
      • act参数指定新的信号处理方式
      • oldact参数输出先前信号的处理方式（如果不为NULL的话）
  • 此程序实现的是捕捉Ctrl+c，在捕捉到之后会执行一系列的操作，其中包含了屏蔽Ctrl+\，即在捕捉到Ctrl+c之后执行对应操作的期间，Ctrl+\信号无效，但若输入了Ctrl+\信号，在操作结束之后仍会生效。
