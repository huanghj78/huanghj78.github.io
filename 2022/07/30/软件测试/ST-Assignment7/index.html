软件测试-Assignment7 等价类测试

Created2022-07-30|软件测试
  1. 构造下述三角形问题的弱健壮的等价类测试用例。
  • 三角形问题：输入三个不超过100的正整数作为三角形的三条边，判断三角形是等边三角形、等腰不等边三角形、完全不等边三角形还是不能构成三角形。

答：

​ 首先，等价类划分通常划分为有效等价类和无效等价类两种情况，若在设计测试用例时同时考虑有效等价类和无效等价类，则称之为健壮性测试，而只覆盖有效等价类的测试用例设计则称为一般等价类测试。其次，根据针对的缺陷数目，又可分为弱等价类测试和强等价类测试，前者是针对单缺陷的等价类测试用例设计，后者则是针对多缺陷的等价类测试用例设计。

​ 故题目要求构造弱健壮的等价类测试用例，即需要考虑有效和无效的等价类且仅需针对单缺陷的用例设计。

有效等价类的划分：

  • R1 = {<a, b, c>: a、b、c构成等边三角形}
  • R2 = {<a, b, c>: a、b、c构成等腰不等边三角形}
  • R3 = {<a, b, c>: a、b、c构成完全不等边三角形}
  • R4 = {<a, b, c>: a、b、c不能构成三角形}

无效等价类的划分：

  • R5 = {<a, b, c>: a > 100}
  • R6 = {<a, b, c>: b > 100}
  • R7 = {<a, b, c>: c > 100}
  • R8 = {<a, b, c>: a <= 0}
  • R9 = {<a, b, c>: b <= 0}
  • R10 = {<a, b, c>: c <= 0}
  • R11 = {<a, b, c>: a不是整数}
  • R12 = {<a, b, c>: b不是整数}
  • R13 = {<a, b, c>: c不是整数}

故最终弱健壮的等价类测试用例设计如下：

测试用例 a b c 期望输出
WR1 5 5 5 等边三角形
WR2 2 2 3 等腰不等边三角形
WR3 3 4 5 完全不等边三角形
WR4 4 1 2 不能构成三角形
WR5 -1 5 5 a超出下界限
WR6 101 5 5 a超出上界限
WR7 5 -1 5 b超出下界限
WR8 5 101 5 b超出上界限
WR9 5 5 -1 c超出下界限
WR10 5 5 101 c超出上界限
WR11 5.1 5 5 a不是整数
WR12 5 5.1 5 b不是整数
WR13 5 5 5.1 c不是整数
